# KCAR 카테고리 트리/판매상품 리스트 UX 개선 기능 정의서

## 기능 개요 (무엇을, 왜)
- 현재 KCAR 화면은 `drct` API가 제공하는 페이징/총건수 정보를 UI에 충분히 반영하지 못하고, 카테고리 트리 정렬 및 선택 유연성도 제한적이다.
- 본 기능은 검색 결과 탐색 효율을 높이기 위해 다음 5가지를 개선한다.
  - 판매상품 리스트 UI 페이징 적용
  - 카테고리 트리 항목 오름차순 정렬(알파벳/가나다)
  - 상위 카테고리 재선택 시 하위 선택 해제(자유 선택)
  - 리스트 총건수 표시 정확화(API 총건수 기준)
  - 상품 카드 핵심 정보 확장(가격 외 식별 정보 노출)

## 사용자 시나리오
1. 사용자는 브랜드/모델군/세부모델/등급을 선택해 판매상품 리스트를 조회한다.
2. 시스템은 `drct` 응답의 `totalCnt`, `pageNo`, `totalPageCnt`, `rows`를 이용해 현재 페이지 결과와 전체 결과 규모를 함께 보여준다.
3. 사용자가 페이지 이동(이전/다음/특정 페이지) 시 동일 필터 조건으로 해당 페이지를 재조회한다.
4. 사용자가 이미 4차까지 선택한 상태에서 2차(예: 기아 > 카니발)를 다시 클릭하면, 선택 상태가 2차까지만 유지되고 3/4차는 해제된다.
5. 사용자는 리스트 각 항목에서 가격뿐 아니라 차량 식별/비교에 필요한 핵심 정보를 확인한다.

## 주요 데이터/인터페이스 개요

### `drct` 응답 필드 (관찰 기준)
- 현재 백엔드 프록시 `/api/kcar/drct-list`는 KCAR 원본 `data`를 그대로 전달한다.
- 확인된 주요 필드:
  - `data.totalCnt`: 조건에 맞는 전체 매물 수
  - `data.pageNo`: 현재 페이지 번호
  - `data.limit`: 페이지 당 건수
  - `data.totalPageCnt`: 전체 페이지 수
  - `data.rows[]`: 현재 페이지 매물 목록

### Unit A - 판매상품 리스트 페이징 UI
- 입력: 선택된 4차 카테고리 코드 + `currentPage`
- 출력: 해당 페이지 `rows` 렌더링 + 페이지 네비게이션
- 요청 파라미터 정책:
  - `pageno`: 선택 페이지 번호
  - `limit`: 기존 기본값 유지(26)
- UI 정책:
  - 첫 페이지/마지막 페이지에서 비활성화 처리
  - 페이지 변경 시 로딩 상태 표시
  - 필터(카테고리) 변경 시 페이지는 1로 초기화

### Unit B - 카테고리 트리 정렬 기준 통일
- 대상: 1~4차 트리 전 단계
- 정렬 기준:
  - 1순위: 표시명 오름차순(알파벳/가나다, ko locale)
  - 2순위: 코드 오름차순(동명이거나 locale 비교 동률 시)
- 기존 `count` 우선 정렬 정책은 제거한다.

### Unit C - 카테고리 자유 선택(상위 재선택)
- 동작 정의:
  - 1차 클릭: 1차 선택 유지, 2~4차 선택/결과 초기화
  - 2차 클릭: 1~2차 선택 유지, 3~4차 선택/결과 초기화
  - 3차 클릭: 1~3차 선택 유지, 4차 선택/결과 초기화
  - 4차 클릭: 기존과 동일하게 리스트 조회
- 동일 노드 재클릭 정책:
  - 이미 선택된 노드를 다시 눌러도 해당 단계 기준으로 하위 선택을 정리한 상태를 보장한다.

### Unit D - 판매상품 총 수량 표시
- 표시 기준:
  - 리스트 헤더의 총건수는 `rows.length`가 아닌 `data.totalCnt`를 우선 사용
  - 예: `판매상품 리스트 (총 32건)`
- 예외 처리:
  - `totalCnt`가 없거나 비정상 값이면 보조적으로 `rows.length` 사용

### Unit E - 판매상품 카드 정보 고도화
- 카드 표시 필드(우선순위):
  - 썸네일: `msizeImgPath` (없으면 `ssizeImgPath` 대체)
  - 차량명: `carWhlNm` (없으면 기존 조합명)
  - 가격: `prc`
  - 차량번호: `cno`
  - 연식: `prdcnYr` 또는 `mfgDt` 기반 표시
  - 주행거리: `milg`
  - 연료/변속기: `fuelNm`, `trnsmsnNm`
- 레이아웃 제약:
  - 항목당 높이가 과도하게 길어지지 않도록 핵심 정보만 최대 2줄 메타로 제한

## 예상 범위
### 포함 범위 (In Scope)
- 프론트 상태에 페이지 상태(`currentPage`, `totalPageCount`, `totalCount`) 추가
- `drct` 요청 시 페이지 파라미터 반영
- 트리 정렬 기준을 오름차순으로 통일
- 상위 재선택 시 하위 선택 해제 규칙 반영
- 리스트 헤더 총건수 표기 수정
- 리스트 카드에 핵심 식별 정보 및 썸네일 렌더링

### 제외 범위 (Out of Scope)
- 페이지 사이즈(`limit`) 사용자 선택 기능
- 무한 스크롤 전환
- 상세 페이지 이동/딥링크
- 서버 측 데이터 가공(정렬/필드 축약) 추가
- 신규 검색 필터(연식, 연료, 가격 구간 등) 도입

## 독립 구현 단위(다음 단계 Plan 입력용)
1. Unit A: `drct` UI 페이징 상태/이벤트/조회 연동
2. Unit B: 트리 1~4차 정렬 정책 변경
3. Unit C: 상위 카테고리 재선택 시 하위 선택 해제
4. Unit D: `totalCnt` 기반 총건수 표기
5. Unit E: 판매상품 카드 핵심 정보 확장

## 완료 기준 (Definition of Done)
- 4차 선택 후 리스트는 페이지 이동이 가능하고, 페이지 전환 시 데이터가 정확히 바뀐다.
- 트리 항목은 단계별로 오름차순 정렬된다.
- 상위 카테고리 재선택 시 하위 단계 선택이 기대대로 정리된다.
- 리스트 헤더 총건수는 API 총건수와 일치한다.
- 리스트 카드는 가격 외 최소 3개 이상의 식별 정보(예: 번호/연식/주행거리)를 표시한다.
