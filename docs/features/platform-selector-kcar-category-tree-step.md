# KCAR 카테고리 트리형 선택 UI 기능 정의서

## 기능 개요 (무엇을, 왜)
- 기존 1~4차 카테고리를 각각 분리된 영역에서 선택하던 UI를 단일 트리 구조 UI로 변경한다.
- 사용자가 1차 카테고리를 선택하면 같은 패널에서 들여쓰기된 하위 2차/3차/4차 항목을 순차적으로 펼쳐 탐색할 수 있어야 한다.
- 카테고리 수가 많아도 한 화면에서 더 많이 보이고 빠르게 선택할 수 있도록 y축 길이(스크롤 영역/행 높이/섹션 고정)를 최적화한다.

## 사용자 시나리오
1. 사용자가 KCAR 화면에 진입한다.
2. 1차 카테고리(브랜드) 목록이 트리 루트 노드로 표시된다.
3. 사용자가 브랜드 노드(1차)를 클릭하면 하위 2차 카테고리(모델군)가 해당 노드 아래 들여쓰기되어 표시된다.
4. 사용자가 2차 노드를 클릭하면 하위 3차(세부 모델)가 추가 들여쓰기되어 표시된다.
5. 사용자가 3차 노드를 클릭하면 하위 4차(등급/트림)가 추가 들여쓰기되어 표시된다.
6. 사용자가 4차 노드를 선택하면 선택 경로(1→2→3→4)가 강조되고 우측 요약 영역에 반영된다.
7. 사용자는 임의의 상위 노드를 다시 클릭해 하위 가지를 접거나 다른 가지를 펼쳐 재탐색할 수 있다.

## 주요 데이터/인터페이스 개요

### 화면 인터페이스
- 좌측 패널
  - 단일 트리 리스트(`tree-view`)로 1~4차 노드를 계층 렌더링
  - 깊이별 들여쓰기(예: depth 1=0px, depth 2=16px, depth 3=32px, depth 4=48px)
  - 노드 상태
    - `collapsed | expanded`
    - `loading | success | empty | error`
    - `selected-path`(현재 선택 경로에 포함된 노드 강조)
  - 노드 우측에 `count` 표시
- 우측 패널
  - 최종 선택 경로 요약(브랜드/모델군/세부 모델/등급)
  - 아직 선택되지 않은 단계 가이드 표시

### 내부 API 인터페이스
- 기존 백엔드 프록시 엔드포인트 재사용
  - `POST /api/kcar/brands` (depth=1)
  - `POST /api/kcar/model-groups` (depth=2)
  - `POST /api/kcar/models` (depth=3)
  - `POST /api/kcar/grades` (depth=4)
- 노드 확장 시점에 필요한 하위 depth만 지연 로딩(lazy load)한다.

### 트리 상태 데이터
- 트리 노드 모델 (예시)
  - `id`: 고유 키 (`brand:001`, `model-group:001:004` 등)
  - `depth`: `1 | 2 | 3 | 4`
  - `label`, `count`
  - `codes`: `{ mnuftrCd?, modelGrpCd?, modelCd?, grdCd? }`
  - `children`: 하위 노드 배열
  - `isExpanded`, `isLeaf`
  - `loadState`: `idle | loading | success | empty | error`
- 화면 상태
  - `selectedPath`: depth별 선택 코드 집합
  - `expandedNodeIds`: 펼쳐진 노드 ID 집합
  - `nodeLoadStateById`: 노드별 하위 로딩 상태
  - `nodeErrorById`: 노드별 오류 메시지
- 표시 정책
  - `count=0` 노드는 모든 depth(1~4차)에서 렌더링 대상에서 제외한다.

### UX/레이아웃 기준 (y축 최적화)
- 트리 패널 높이: `calc(100vh - 상단 여백)` 기반 고정, 패널 내부 스크롤 사용
- 행 높이: 현재 대비 축소(컴팩트)하되 터치 영역은 유지
- 스크롤 편의
  - 선택 노드가 보이지 않으면 자동 스크롤 보정
  - 긴 목록에서 스크롤 위치 보존(접기/펼치기 시 급격한 점프 방지)
- 표시 밀도
  - 레벨 뱃지/아이콘은 최소화하고 텍스트+카운트 중심
  - 필요 시 depth별 폰트/간격 차등으로 정보량 증가

## 예상 범위
### 포함 범위 (In Scope)
- 1~4차 카테고리의 트리형 UI 전환
- 노드 단위 lazy load 및 에러/빈 상태 처리
- 들여쓰기 기반 계층 표현, 펼침/접힘 인터랙션
- 선택 경로 강조 및 우측 요약 연동
- 대량 노드 대응을 위한 y축 스크롤/밀도 최적화
- `count=0` 카테고리 항목 비노출 처리

### 제외 범위 (Out of Scope)
- 매물 리스트(`drct`) 연동 및 검색 결과 화면 전환
- 다중 선택(체크박스형) 트리
- 사용자별 트리 상태 저장(즐겨찾기/최근 선택)
- 서버 API 계약 변경(기존 엔드포인트 유지)
