# 기능 정의 — 차량 상세 조회 (Car Detail Lookup)

> 작성일: 2026-03-01  
> 단계: 1단계 (Feature Spec)  
> 상태: 초안 (피드백 대기)

---

## 기능 개요

### 무엇을
엔카(encar.com) 차량 상세 페이지 URL을 입력받아, 해당 차량의 기본 정보와 보험이력 정보를 수집하여 사용자에게 깔끔하게 시각화해서 보여주는 기능.

### 왜
- 엔카/케이카는 영업용 이력, 소유주 변경 횟수 등 구매자에게 중요한 필터를 의도적으로 제공하지 않음
- 사용자가 URL 하나만 붙여넣으면 해당 차량의 핵심 품질 지표를 한눈에 파악할 수 있게 하는 것이 이 프로젝트의 첫 번째 핵심 가치

---

## 서비스 전체 구조 (Big Picture)

이 기능은 아래 전체 서비스 흐름의 **가장 첫 번째 구현 단위**이다.

```
홈화면
 ├── [엔카로 검색] 버튼  ← 현재 활성
 └── [케이카로 검색] 버튼  ← 준비중 (disabled)
      ↓
카테고리 선택 (미래 기능)
 1차: 국산차 / 수입차
 2차: 브랜드 (현대, 기아, BMW 등)
 3차: 차종 (아반떼, 소나타 등)
 4차: 세대 (7세대, 8세대 등)
      ↓
차량 목록 (미래 기능)
      ↓
차량 상세 조회 ← ✅ 지금 만드는 기능 (URL 직접 입력 방식으로 시작)
```

---

## 사용자 시나리오

1. 사용자가 홈화면에서 **[엔카로 검색]** 버튼을 클릭한다.
2. 엔카 차량 URL 입력창이 나타난다.
3. 사용자가 엔카 차량 상세 페이지 URL을 붙여넣는다.
   - 예시: 'https://fem.encar.com/cars/detail/41471109'
   - 설명: 위 예시에서 41471109는 vehicleId 로 자동차 식별번호다.
4. [조회하기] 버튼을 누르면 해당 차량의 정보가 화면에 표시된다.

| 지역 | 매물 등록 지역 |

| 대표 이미지 | 차량 사진 |

### 보험이력 정보 (카히스토리 / 보험개발원 연동 여부 확인 필요)
| 항목 | 설명 | 중요도 |
|------|------|--------|
| 소유자 변경 횟수 | 몇 번 손을 탔는지 | ⭐⭐⭐ |
| 영업용 사용 이력 | 택시/렌트/법인 여부 | ⭐⭐⭐ |
| 사고 이력 | 내 차 피해 / 상대 차 피해 구분 | ⭐⭐⭐ |
| 전손 이력 | 전손 처리 여부 | ⭐⭐⭐ |
| 침수 이력 | 침수 처리 여부 | ⭐⭐⭐ |

## 범위 (Scope)

### ✅ In Scope (이번에 만들 것)
- 홈화면: 엔카/케이카 선택 버튼 2개
- 엔카 차량 URL 입력 → 차량 기본 정보 조회 및 표시
- 보험이력 정보 조회 및 표시 (엔카 내부 보험이력 API 활용 검토)
- 차량 정보 UI (깔끔한 카드형 레이아웃)

### ⛔ Out of Scope (이번에 만들지 않을 것)
- 케이카 연동
- 카테고리 탐색 (국산/수입 → 브랜드 → 차종 → 세대)
- 차량 목록 페이지
- 품질 점수 계산 로직
- 로그인/회원 기능
- 즐겨찾기/저장 기능

---

## 기술적 고려사항 (메모)

- 엔카는 공식 오픈 API가 없으므로 **내부 API(비공개 REST API) 호출 방식**으로 접근 예정
  - 실제 호출 가능 여부 및 응답 구조는 2단계(Plan) 수립 전 반드시 테스트 필요
- 보험이력 데이터는 엔카가 자체 제공하는 경우와 카히스토리 등 외부 서비스 연동 여부 구분 필요
- CORS 이슈 가능성 있음 → 서버사이드(백엔드 프록시) 경유 필요 가능성 높음
