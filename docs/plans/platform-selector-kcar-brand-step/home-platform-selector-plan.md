# Sub Task Plan: 첫 화면 플랫폼 선택 UI

## 목표
- 첫 화면에서 `encar`/`kcar` 버튼을 제공하고, 명세된 상태/동작을 반영한다.

## 작업 범위
- 플랫폼 선택 카드/섹션 UI
- 버튼 상태 및 클릭 이벤트 연결
- 접근성 속성(비활성/포커스) 적용

## 구현 순서
1. 플랫폼 선택 컴포넌트 생성
2. `encar` 버튼을 `disabled`로 고정하고 `제공예정` 표기 적용
3. `kcar` 버튼 클릭 시 `/kcar` 라우팅 연결
4. 키보드 포커스/스크린리더 상태 점검

## 완료 기준
- `encar`는 클릭 불가, 스타일상 비활성으로 구분된다.
- `kcar` 클릭 시 KCAR 페이지로 이동한다.
- 마우스/키보드 상호작용이 접근성 요구를 만족한다.

## 의존성
- Task 1 라우팅 골격

## 리스크
- 디자인 시스템 부재로 버튼 스타일 기준이 임시값으로 들어갈 수 있음.
