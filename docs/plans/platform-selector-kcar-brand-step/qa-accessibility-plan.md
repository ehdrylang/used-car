# Sub Task Plan: QA 및 접근성 검증

## 목표
- 기능 명세의 Acceptance Criteria를 기준으로 동작/접근성/회귀를 검증한다.

## 작업 범위
- 단위/컴포넌트 테스트
- 수동 시나리오 테스트
- 접근성 핵심 체크(키보드 이동, 비활성 전달)

## 검증 항목
1. 첫 화면에 `encar`, `kcar` 버튼 표시
2. `encar` 비활성 및 클릭 무효
3. `kcar` 클릭 시 KCAR 화면 이동
4. 브랜드 목록 국산/수입 구분 노출
5. 브랜드명 + count 표시
6. 브랜드 선택 상태 반영
7. API 실패 시 에러/재시도 제공
8. 빈 데이터 메시지 노출
9. 모델 선택 UI 미노출 + 안내 문구 노출

## 완료 기준
- 9개 AC 항목이 모두 통과한다.
- 주요 테스트 결과가 문서 또는 PR 코멘트로 기록된다.
- 구현 범위 밖 기능(모델 선택/매물조회)이 섞이지 않았다.

## 의존성
- Task 2, Task 4 구현 결과

## 리스크
- 실제 API 기반 테스트 시 외부 의존성으로 결과가 불안정할 수 있음.
