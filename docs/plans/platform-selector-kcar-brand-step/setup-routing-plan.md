# Sub Task Plan: 프로젝트 셋업 및 라우팅 골격

## 목표
- `home`(플랫폼 선택)와 `kcar`(브랜드 선택) 화면으로 이동 가능한 최소 라우팅 구조를 준비한다.

## 작업 범위
- 앱 엔트리/라우터 설정
- 페이지 컴포넌트 파일 생성
- 공통 레이아웃(헤더/메인 컨테이너) 최소 구조 생성

## 구현 순서
1. 라우터에 `/` -> Home 페이지, `/kcar` -> KCAR 페이지 매핑
2. 각 페이지에 임시 헤더 및 본문 영역 배치
3. 추후 UI 구현을 위한 컴포넌트 배치 포인트 정의

## 완료 기준
- URL 전환으로 두 화면이 정상 표시된다.
- 브라우저 새로고침 시 라우팅이 깨지지 않는다.
- 이후 Task에서 UI/상태를 추가할 수 있는 구조가 준비된다.
