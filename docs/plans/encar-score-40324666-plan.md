# ENCAR 차량 점수 노출 계획

## 요구사항 요약
- `SCORING.md` 기준으로 ENCAR 차량 점수를 계산한다.
- 우선 대상은 차량 ID `40324666`이며, 추후 다른 ENCAR 차량 ID도 재사용 가능해야 한다.
- 점수와 근거를 리스트 형태 UI에 노출한다.

## 구현 범위
- 백엔드에 `POST /api/encar/vehicle-score` 추가
- ENCAR `record`, `inspection`, `diagnosis` 응답을 조합해 점수 계산
- 프론트 홈에서 `encar` 선택 가능하도록 변경
- ENCAR 차량 ID 입력 + 조회 UI 추가
- 평가 결과를 카드형 리스트로 노출

## 점수 매핑
- 용도이력: 영업용/용도변경 감점
- 사고/보험이력: 보험 미가입 기간, 무사고/전손/침수, 보험 수리 평균금액 반영
- 1인소유: 소유주 변경 횟수와 변경 간격 반영
- 판매자 정보: 현재 미연동이므로 중립 처리
- 주행거리: 최초 등록일 대비 연평균 주행거리 반영
- 연식: 현재 연도 기준 차령 반영
- 성능점검상: 리콜 미이행 제외, 주요 골격, 본네트/휀더, 누유 반영
- 본문: 현재 ENCAR 상세 본문 미연동이므로 중립 처리

## 수용 기준
- `40324666` 조회 시 점수 총점과 근거 목록이 보여야 한다.
- 성능점검/진단 API가 404여도 페이지는 깨지지 않고 가능한 범위만 평가해야 한다.
- 리콜 미이행 차량은 총점 대신 평가 제외 사유가 보여야 한다.
