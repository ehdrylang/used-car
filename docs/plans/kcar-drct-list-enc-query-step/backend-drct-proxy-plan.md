# Sub Task Plan: 백엔드 `drct` 프록시 엔드포인트 구현

## 목표
- 프론트가 생성한 `enc`를 받아 KCAR `/bc/search/list/drct`로 전달하는 프록시 API를 추가한다.

## 작업 범위
- `POST /api/kcar/drct-list` 라우트 추가
- 요청 바디 검증 (`enc` 필수/문자열/빈 값 불가)
- 업스트림 호출 및 타임아웃 처리
- 표준 성공/실패 응답 포맷 적용

## 구현 순서
1. `enc` 검증 파서 함수 추가
2. 업스트림 `drct` 호출 함수 추가
3. 라우트 등록 및 메서드 제한(`POST`)
4. 응답 정규화(`meta.source`, `meta.fetchedAt`)
5. API 계약 문서 업데이트

## 완료 기준
- 정상 `enc` 입력 시 `200`으로 리스트 데이터 응답
- 잘못된 `enc` 입력 시 `400 BAD_REQUEST`
- 업스트림 장애/지연 시 기존 에러 코드 체계 유지

## 의존성
- Task 1 프론트 `enc` 생성

## 리스크
- 업스트림 응답 구조 변동 시 정규화 로직 수정이 필요할 수 있다.
