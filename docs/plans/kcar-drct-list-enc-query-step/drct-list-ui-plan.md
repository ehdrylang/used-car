# Sub Task Plan: 판매상품 리스트 UI 렌더링

## 목표
- 우측 패널에 카테고리 선택 결과에 따른 `drct` 판매상품 리스트를 명확히 표시한다.

## 작업 범위
- 상태별 UI 구성
  - `loading`: 스켈레톤/로딩 메시지
  - `error`: 오류 문구 + 재시도
  - `empty`: 결과 없음 안내
  - `success`: 차량 리스트 카드/행 렌더링
- 기존 선택 요약 UI와 충돌 없이 배치
- y축 스크롤/가독성 최적화

## 구현 순서
1. 리스트 컨테이너와 상태 메시지 컴포넌트 추가
2. 응답 데이터에서 최소 표시 필드 추출
3. 카드/행 스타일 정의
4. 모바일에서 레이아웃 깨짐 여부 보정
5. 선택 요약 영역과 동시 노출 UX 조정

## 완료 기준
- 4차 선택 이후 사용자에게 조회 결과가 즉시 보인다.
- 상태 전환(로딩→성공/오류/빈)이 자연스럽다.
- 기존 카테고리 조작 UI를 방해하지 않는다.

## 의존성
- Task 3 조회 흐름 연결

## 리스크
- 응답 필드가 많아 과도한 정보 노출 시 가독성이 떨어질 수 있다.
