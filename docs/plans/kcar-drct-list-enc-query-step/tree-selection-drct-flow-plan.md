# Sub Task Plan: 트리 선택과 `drct` 조회 흐름 연결

## 목표
- 카테고리 트리 선택 흐름을 유지하면서 4차 선택 완료 시 판매상품 조회를 연결한다.

## 작업 범위
- 4차 선택 이벤트에 `drct` 조회 트리거 추가
- 조회 조건 생성 (`mnuftrCd,modelGrpCd,modelCd,grdCd`)
- 상위 카테고리 변경 시 기존 리스트/조회 상태 초기화
- 중복 요청 방지 및 최신 선택 기준 반영

## 구현 순서
1. 리스트 상태(`drctLoadState`, `drctRows`, `drctError`) 추가
2. 4차 선택 시 `buildParam -> enc -> API 호출` 연결
3. 상위 노드 재선택 시 리스트 상태 초기화 로직 추가
4. 요청 시퀀스 토큰으로 응답 역전 방지
5. 재시도 버튼 동작 연결

## 완료 기준
- 4차 선택 시 자동으로 조회 요청이 실행된다.
- 상위 선택이 바뀌면 이전 결과가 남지 않는다.
- 오래된 응답이 최신 화면을 덮어쓰지 않는다.

## 리스크
- 사용자가 빠르게 선택 변경 시 요청 과다가 발생할 수 있다.
