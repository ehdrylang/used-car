# Sub Task Plan: `drct` 조회 QA 및 접근성 검증

## 목표
- 신규 `enc` 기반 `drct` 조회 기능이 기존 카테고리 기능을 깨지 않고 동작하는지 검증한다.

## 작업 범위
- 기능 QA
  - 4차 선택 시 조회 실행
  - 상위 선택 변경 시 리스트 초기화
  - 오류/재시도 동작
- 회귀 QA
  - 홈→KCAR 진입
  - 트리 확장/선택/요약 기능 유지
  - `count=0` 비노출 유지
- 접근성 QA
  - 키보드 선택 흐름
  - 상태 메시지(`aria-live`, `role=alert`) 점검

## 검증 시나리오
1. 4차 선택 완료 시 `drct` 요청이 1회 발생하는지 확인
2. 동일 4차 재선택 시 중복 요청 정책(허용/차단) 확인
3. 다른 브랜드 가지 선택 시 기존 리스트가 사라지고 새 기준으로 조회되는지 확인
4. API 실패 시 오류 안내와 재시도 버튼 동작 확인
5. 트리에서 `count=0` 항목이 여전히 표시되지 않는지 확인
6. 모바일 화면에서 트리+리스트 스크롤 동선 확인

## 완료 기준
- 신규 기능이 기존 카테고리 UX를 회귀 없이 확장한다.
- 주요 오류 시나리오에서 사용자가 복구 가능하다.
- 데스크톱/모바일에서 치명적 UI 이슈가 없다.

## 의존성
- Task 1~4 완료

## 리스크
- 실데이터 변동으로 결과 건수 기준 검증값이 수시로 바뀔 수 있다.
