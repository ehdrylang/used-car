# Sub Task Plan: 트리 렌더링 및 y축 최적화 UI

## 목표
- 한 패널에서 1~4차 카테고리를 들여쓰기 트리로 표시하고, 많은 항목도 빠르게 탐색 가능한 y축 레이아웃을 구성한다.

## 작업 범위
- 단일 트리 리스트 렌더링 함수 구현
- depth별 들여쓰기/아이콘/활성 경로 스타일 적용
- 패널 고정 높이 + 내부 스크롤 구성
- 행 높이/간격 컴팩트화 및 모바일 대응

## 구현 순서
1. 트리 노드 재귀(또는 평탄화) 렌더 방식 선택
2. depth별 left padding, 타이포, 경로 강조 스타일 적용
3. 패널 `max-height`/`overflow` 조정으로 y축 스크롤 구조 구성
4. 노드 선택 시 가시 영역 자동 보정(scroll into view) 적용
5. 모바일에서 행 높이/터치 영역 균형 조정

## 완료 기준
- 1~4차 노드가 한 화면에서 계층 구조로 자연스럽게 읽힌다.
- 기존 분리 영역 대비 더 많은 항목을 표시할 수 있다.
- 스크롤 점프/겹침 없이 안정적으로 탐색된다.
