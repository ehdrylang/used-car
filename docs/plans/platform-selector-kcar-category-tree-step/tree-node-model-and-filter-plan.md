# Sub Task Plan: 트리 노드 모델 및 count 필터링 정책

## 목표
- 트리형 카테고리 렌더링에 필요한 공통 노드 모델을 정의하고, `count=0` 비노출 정책을 확정한다.

## 작업 범위
- depth별 노드 스키마 정의(1~4차)
- 노드 고유 ID 규칙 정의
- 트리 데이터 정규화/정렬 기준 정의
- `count > 0` 필터링 공통 함수 설계

## 구현 순서
1. depth별 노드 타입(brand/modelGroup/model/grade) 정리
2. 공통 노드 필드(`id`, `depth`, `label`, `count`, `codes`) 매핑 규칙 정의
3. `count <= 0` 제거 함수 작성 위치와 적용 시점(응답 직후) 확정
4. 빈 노드 처리 메시지 규칙(표시 가능한 항목 없음) 정의

## 완료 기준
- 모든 depth에서 동일한 노드 구조로 렌더링 가능하다.
- count 필터링 기준이 코드와 문서에서 일관된다.
- 필터링 이후에도 선택/확장 상태 키 충돌이 없다.

## 의존성
- 없음

## 리스크
- 필터링 시점이 늦으면 UI가 잠시 0건 항목을 표시할 수 있다.
