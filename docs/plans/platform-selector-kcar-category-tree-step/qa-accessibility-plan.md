# Sub Task Plan: 트리 UI QA 및 접근성 검증

## 목표
- 트리형 카테고리 UI와 `count=0` 미노출 정책이 요구사항대로 동작하는지 기능/접근성/회귀 관점에서 검증한다.

## 작업 범위
- 기능 QA
  - 노드 확장/접기/선택/재시도
  - `count=0` 항목 비노출 검증
  - 상위 선택 변경 시 하위 경로 초기화
- 접근성 QA
  - 키보드 탐색(포커스 이동, Enter/Space 선택)
  - `aria-expanded`, `aria-selected`, `aria-live` 검증
- UI QA
  - 데스크톱/모바일 스크롤 사용성 확인

## 검증 시나리오
1. 브랜드 노드 확장 시 2차 노드가 들여쓰기되어 표시되는지 확인
2. 2차/3차 확장 시 각각 3차/4차 노드가 연결 표시되는지 확인
3. API 응답에 `count=0` 항목이 있어도 트리에서 보이지 않는지 확인
4. 노드 오류 발생 시 해당 노드에서만 재시도 가능한지 확인
5. 다른 브랜드 가지로 이동 시 이전 하위 선택 경로가 초기화되는지 확인
6. 모바일 뷰에서 긴 목록 스크롤이 끊기지 않는지 확인

## 완료 기준
- 트리형 탐색 시나리오가 1~4차까지 정상 재현된다.
- `count=0` 항목 미노출 정책이 전 depth에서 충족된다.
- 키보드 접근성과 반응형 동작에 치명적 이슈가 없다.

## 의존성
- Task 1~4 완료

## 리스크
- 실제 데이터 변동으로 QA 재현 결과가 수시로 달라질 수 있다.
