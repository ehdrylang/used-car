# Sub Task Plan: 선택 경로/재탐색 상호작용 및 요약 연동

## 목표
- 트리 탐색 중 선택 경로와 펼침 상태를 명확히 보여주고, 우측 요약과 항상 동기화한다.

## 작업 범위
- 선택 경로 상태(`selectedPath`) 관리
- 접기/펼치기 시 하위 선택 유지/초기화 정책 정의
- 우측 요약 카드와 실시간 동기화
- 단계 미완료 시 가이드 문구 표시

## 구현 순서
1. depth별 선택 코드 경로 업데이트 로직 작성
2. 상위 노드 재선택 시 하위 경로 초기화 규칙 반영
3. 트리 경로 강조 스타일(ancestor/current) 적용
4. 우측 요약에 1~4차 선택값 및 count 반영
5. 최종 선택(4차) 완료 상태 문구 분기 처리

## 완료 기준
- 트리에서 보이는 활성 경로와 우측 요약 값이 항상 동일하다.
- 재탐색(다른 가지 선택) 시 하위 경로가 정확히 갱신된다.
- 미선택 depth에 대한 사용자 안내가 명확하다.

## 의존성
- Task 2, Task 3

## 리스크
- 선택 유지 정책이 일관되지 않으면 사용자가 현재 기준을 오해할 수 있다.
