# Sub Task Plan: 판매상품 리스트 페이징 연동 (Unit A)

## 목표
- `drct` API가 제공하는 페이지 정보를 UI와 요청 파라미터에 연결해 화면에서 실제 페이징 조회가 가능하도록 한다.

## 작업 범위
- 상태 확장
  - `currentPage`, `totalPageCount`, `totalCount`, `pageLimit` 상태 추가
- 조회 로직 확장
  - `fetchDrctList(..., pageNo)` 시그니처 확장
  - `createDrctPlainParam`에 `pageno` 반영
- UI 확장
  - 이전/다음 버튼, 현재 페이지/전체 페이지 표시
  - 필요 시 인접 페이지 번호 버튼(예: 최대 5개) 제공
- 초기화 규칙
  - 카테고리 변경 시 `currentPage=1`, 페이징 메타 초기화

## 구현 순서
1. `appState`에 페이징 상태를 추가한다.
2. `fetchDrctList`가 응답의 `totalCnt`, `pageNo`, `totalPageCnt`, `limit`를 저장하도록 확장한다.
3. `kcar-enc.js`에서 `createDrctPlainParam`의 `pageno`를 동적으로 받게 수정한다.
4. 리스트 하단에 페이지 네비게이션 렌더링 함수를 추가한다.
5. 페이지 버튼 이벤트를 바인딩하고 경계값(1페이지/마지막 페이지)을 처리한다.

## 검증 시나리오
1. 4차 선택 직후 1페이지가 조회되고 현재 페이지가 `1`로 표시된다.
2. 다음 페이지 클릭 시 `pageno=2` 요청이 발생하고 결과가 바뀐다.
3. 마지막 페이지에서 다음 버튼이 비활성화된다.
4. 다른 카테고리로 전환 시 페이지가 1로 리셋된다.

## 완료 기준
- UI에서 페이지 이동이 가능하고, 이동한 페이지와 응답 `pageNo`가 일치한다.
- 페이징 메타(`totalCount`, `totalPageCount`)가 리스트 헤더/네비게이션과 일관된다.
- 카테고리 변경 시 이전 페이지 상태가 누수되지 않는다.
