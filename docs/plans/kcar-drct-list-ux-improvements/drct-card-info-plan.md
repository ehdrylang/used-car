# Sub Task Plan: 판매상품 카드 핵심 정보 확장 (Unit E)

## 목표
- 가격 외 식별 가능한 핵심 정보를 카드에 함께 노출해 리스트 구분성과 비교 효율을 높인다.

## 작업 범위
- 카드 필드 확장
  - 이미지: `msizeImgPath` 우선, `ssizeImgPath` 대체
  - 차량명: `carWhlNm` 우선
  - 메타: `cno`, `prdcnYr|mfgDt`, `milg`, `fuelNm`, `trnsmsnNm`
- 표시 규칙
  - 누락 필드는 `-` 또는 해당 항목 숨김으로 처리
  - 메타 영역은 최대 2줄로 제한
- 스타일 확장
  - 썸네일 + 텍스트 2열 카드
  - 모바일에서는 세로 적층 레이아웃

## 구현 순서
1. 카드 렌더 함수에 필드 추출/포맷 유틸을 추가한다.
2. 우선순위 기반 fallback 규칙을 적용한다.
3. CSS에서 카드 레이아웃과 텍스트 줄바꿈/말줄임 규칙을 추가한다.
4. 이미지 로드 실패 시 대체 표시를 적용한다.

## 검증 시나리오
1. 정상 데이터에서 이미지/차량명/가격/번호/주행거리가 함께 보인다.
2. 일부 필드 누락 데이터에서도 카드가 깨지지 않는다.
3. 모바일 화면에서 카드 길이가 과도하게 늘어나지 않는다.

## 완료 기준
- 가격 외 최소 3개 이상 핵심 식별 정보가 일관되게 노출된다.
- 정보량 증가에도 리스트 가독성과 스크롤 사용성이 유지된다.

## 의존성
- 없음 (Unit A/D와 병행 가능)

## 리스크
- 카드 정보가 과다하면 한 화면 정보밀도가 낮아질 수 있어 필드 우선순위 유지가 필요하다.
