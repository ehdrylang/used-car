# Sub Task Plan: 상위 카테고리 재선택 흐름 개선 (Unit C)

## 목표
- 이미 하위까지 선택된 상태에서도 상위 카테고리를 자유롭게 재선택할 수 있도록 선택 해제/초기화 규칙을 명확히 적용한다.

## 작업 범위
- 클릭 정책 정교화
  - 1차 클릭 시 2~4차 상태 초기화
  - 2차 클릭 시 3~4차 상태 초기화
  - 3차 클릭 시 4차 상태 초기화
- 동일 노드 재클릭 정책
  - 기존 "동일 노드면 fetch 생략" 중심 로직을 "하위 정리 보장" 중심으로 보완
- 트리 노출 정책
  - 재선택 후 해당 단계까지만 active/path가 유지되도록 렌더링 일치화

## 구현 순서
1. 브랜드/모델군/모델 클릭 핸들러의 분기 조건을 재정의한다.
2. `resetFromBrandSelection`, `resetFromModelGroupSelection`, `resetFromModelSelection` 호출 시점을 조정한다.
3. 동일 코드 재클릭 시에도 하위 상태 정리가 필요한 케이스를 반영한다.
4. 필요 시 상위 재선택 이후 하위 목록 재조회 정책(즉시/지연)을 명시한다.

## 검증 시나리오
1. 4차까지 선택 후 2차 노드 클릭 시 3/4차 선택이 해제된다.
2. 3차까지 선택 후 1차 노드 클릭 시 2/3/4차 선택이 해제된다.
3. 트리에서 active/path 하이라이트가 선택 단계와 정확히 일치한다.
4. 재선택 후 `drct` 결과가 이전 조건으로 남아있지 않는다.

## 완료 기준
- 사용자 요구대로 상위 단계에서 자유롭게 재탐색이 가능하다.
- 선택 상태/조회 상태/렌더링 상태가 단계별로 일관된다.

## 의존성
- Unit A와 결합 시 검증이 가장 명확하지만, 독립 구현 가능

## 리스크
- 재선택 시 불필요한 재조회가 늘어날 수 있어 요청 최적화 정책이 필요하다.
