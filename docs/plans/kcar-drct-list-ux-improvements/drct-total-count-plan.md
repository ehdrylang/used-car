# Sub Task Plan: 판매상품 총 수량 표기 정확화 (Unit D)

## 목표
- 리스트 헤더 건수를 현재 페이지 길이(`rows.length`)가 아닌 전체 건수(`totalCnt`) 중심으로 표시한다.

## 작업 범위
- 데이터 모델 확장
  - `drct` 응답에서 `totalCnt`를 상태로 저장
- 표시 로직 변경
  - 헤더 텍스트를 `판매상품 리스트 (총 N건)`으로 통일
- 예외 처리
  - `totalCnt` 누락/비정상 시 `rows.length` fallback

## 구현 순서
1. `fetchDrctList` 성공 시 `totalCnt`를 파싱해 상태에 반영한다.
2. 리스트 헤더 렌더링에서 표시 건수 계산 함수를 분리한다.
3. `empty` 상태에서도 총건수 표기가 일관되도록 규칙을 정리한다.

## 검증 시나리오
1. `limit=26`, `totalCnt=32`일 때 헤더가 `총 32건`으로 표시된다.
2. 2페이지에서도 헤더 총건수가 동일하게 유지된다.
3. 응답 이상(`totalCnt` 없음) 시 헤더가 안전하게 fallback된다.

## 완료 기준
- 총건수 표기가 API 기준과 일치한다.
- 페이지별 행 수와 총건수의 의미가 UI에서 혼동되지 않는다.

## 의존성
- Unit A (페이징 메타 수집)와 함께 구현 시 가장 자연스럽다.

## 리스크
- 응답 스키마가 변경되면 총건수 파싱 실패 가능성이 있어 방어코드가 필요하다.
