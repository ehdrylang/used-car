# Sub Task Plan: 카테고리 트리 오름차순 정렬 (Unit B)

## 목표
- 1~4차 카테고리 트리 노드를 `count` 우선이 아닌 이름 오름차순(알파벳/가나다)으로 정렬한다.

## 작업 범위
- 정렬 유틸 변경
  - 기존 `count` 우선 정렬 제거
  - 이름(`pathNm`, `modelGrpNm`, `modelNm`, `grdNm`) 기준 locale 정렬 적용
- 동률 처리
  - 코드 필드(`mnuftrCd`, `modelGrpCd`, `modelCd`, `grdCd`) 오름차순으로 deterministic 정렬
- 적용 대상
  - 브랜드 목록
  - 모델군 목록
  - 세부 모델 목록
  - 등급/트림 목록

## 구현 순서
1. 공통 정렬 함수(이름+코드 기준)를 설계한다.
2. 단계별 목록 생성 지점에서 해당 정렬 함수를 사용한다.
3. `count=0` 비노출 정책은 유지한다.
4. 국산/수입 탭 각각에서 정렬 결과를 검증한다.

## 검증 시나리오
1. 동일 단계 목록이 항상 이름 오름차순으로 출력된다.
2. 한글/영문 혼합 데이터에서도 정렬이 안정적으로 유지된다.
3. 데이터 재조회 후에도 순서가 일관된다.

## 완료 기준
- 트리 전 단계가 요구한 오름차순 정책으로 통일된다.
- 기존 선택/로딩/오류 처리 흐름은 영향받지 않는다.

## 의존성
- 없음 (독립 구현 가능)

## 리스크
- locale 비교 결과가 브라우저별로 미세 차이가 있을 수 있어 tie-breaker 코드 정렬이 필요하다.
