# MVP 서비스 개발 계획

## 서비스 개요

엔카 API를 활용해 중고차 구매자가 **꼭 필요한 정보만** 빠르게 확인할 수 있는 웹 서비스를 만든다.  
차량 ID를 입력하면 신뢰도 핵심 지표를 한눈에 보여주는 것이 목표다.

---

## 핵심 정보 (MVP 범위)

차량 1대에 대해 아래 4가지 API를 호출해 필수 정보만 추출하여 표시한다.

| 항목 | API | 표시할 정보 |
|------|-----|------------|
| 판매자 신뢰도 | `GET /v1/readside/user/{userId}` | 엔카 등록일 (`joinedDatetime`) → 활동 연수 |
| 차량 이력 | `GET /v1/readside/record/vehicle/{vehicleId}/open` | 소유주 변경 횟수, 보험 처리 이력 건수·금액 |
| 수리 이력 | `GET /v1/readside/diagnosis/vehicle/{vehicleId}` | 프레임 손상 여부 (교환·판금 부위 목록) |
| 성능 점검 | `GET /v1/readside/inspection/vehicle/{vehicleId}` | 주행거리(km), 변속기 유형, 색상, 수리부위 목록 |

> **이상 감지**: 진단 API의 수리 부위 ≠ 성능점검 API의 수리 부위인 경우 "불일치 주의" 경고 표시

```
  · 주행거리: N km

  · 색상: OOO
────────────────────────────────
```

## 기술 스택

- **Frontend**: Next.js (React) + Tailwind CSS
- **Backend(API Route)**: Next.js API Routes — 엔카 API 호출은 서버 사이드에서 처리 (CORS 우회)
- **배포**: Vercel

---

## 개발 단계

### Phase 1 — 백엔드 API 연동
1. `GET /api/car?vehicleId=&userId=` 엔드포인트 구현
2. 엔카 API 4개 병렬 호출 (`Promise.all`)
3. 필요한 필드만 추출해 단일 JSON으로 응답

### Phase 2 — 프론트엔드 UI
1. 차량 ID / 판매자 ID 입력 폼
2. 조회 결과를 카드 형태로 표시
3. 불일치 경고 배지 처리

### Phase 3 — 안정화
1. 로딩 / 에러 상태 처리
2. 모바일 대응 (반응형)
