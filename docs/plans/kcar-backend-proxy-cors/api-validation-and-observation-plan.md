# Sub Task Plan: API 검증 및 관찰 문서화

## 목표
- 구현된 백엔드 API의 요청/응답 계약을 실호출 기준으로 검증하고 문서화한다.

## 작업 범위
- 로컬 API 호출 예시 수집
- 정상/오류 케이스 응답 저장
- 필드별 의미 및 프론트 매핑 포인트 주석화
- 문서 파일 업데이트

## 구현 순서
1. 정상 케이스 호출 및 응답 캡처
2. 잘못된 바디로 `BAD_REQUEST` 검증
3. 업스트림 실패/타임아웃 시뮬레이션 검증
4. `docs/apis/kcar-backend-api-contract.md` 작성

## 완료 기준
- API 계약 문서가 최신 구현 기준으로 존재한다.
- 프론트 연동에 필요한 필드와 에러 코드가 확정된다.

## 의존성
- Task 3 KCAR 브랜드 프록시 구현

## 리스크
