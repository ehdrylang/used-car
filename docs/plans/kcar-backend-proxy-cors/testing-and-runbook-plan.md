# Sub Task Plan: 테스트 및 운영 런북 정리

## 목표
- 백엔드 프록시 기능을 안정적으로 유지하기 위한 테스트와 실행 가이드를 정리한다.

## 작업 범위
- 단위/통합 테스트 케이스 정의
- 로컬 실행/배포 전 점검 절차 문서화
- 장애 시 확인 포인트(로그/에러 코드) 정리
- README 실행 가이드 업데이트

## 테스트 항목
1. 정상 브랜드 조회 응답
2. `sellType` 기본값 처리
3. 잘못된 바디 요청 처리(`BAD_REQUEST`)
4. 업스트림 비정상 응답 처리(`UPSTREAM_ERROR`)
5. 타임아웃 처리(`TIMEOUT`)
6. CORS preflight 동작

## 완료 기준
- 핵심 시나리오 테스트가 자동화 또는 반복 가능한 스크립트로 정리된다.
- 신규 개발자가 문서만으로 로컬 실행/검증 가능하다.

## 의존성
- Task 1~5

## 리스크
- 테스트에서 업스트림 실호출 의존이 크면 결과가 불안정할 수 있어 모킹 전략이 필요.
