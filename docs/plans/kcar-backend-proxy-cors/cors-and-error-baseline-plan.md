# Sub Task Plan: CORS 및 에러 처리 베이스라인

## 목표
- CORS 정책과 표준 에러 응답 구조를 서버 공통 계층으로 구축한다.

## 작업 범위
- 허용 origin 화이트리스트 설정
- preflight(`OPTIONS`) 응답 처리
- 공통 에러 핸들러 구현
- 요청 ID/응답시간 기본 로깅 적용

## 구현 순서
1. CORS 미들웨어 설정
2. 에러 객체 표준화(`code`, `message`, `status`)
3. 전역 에러 핸들러 연결
4. 예외 상황 응답 포맷 검증

## 완료 기준
- 허용된 origin은 정상 통신, 비허용 origin은 차단된다.
- 예외 발생 시 표준 에러 JSON이 반환된다.

## 의존성
- Task 1 백엔드 부트스트랩

## 리스크
- 개발/운영 origin 분리 누락 시 실제 배포 환경에서 장애 가능성.
