# Sub Task Plan: 프론트 연동 경로 전환

## 목표
- 프론트엔드가 외부 KCAR API 대신 내부 백엔드 API를 호출하도록 전환한다.

## 작업 범위
- 프론트 API URL 변경 (`/api/kcar/brands`)
- 개발 환경에서 백엔드 주소 설정
- 에러 메시지/재시도 동작 유지
- CORS 문제 해소 검증

## 구현 순서
1. 프론트 API 엔드포인트 상수 교체
2. 환경변수 또는 프록시 경로 반영
3. 수동 테스트로 브랜드 로딩/선택 확인
4. 회귀 테스트(홈 화면, 브랜드 탭, 재시도)

## 완료 기준
- 브라우저 콘솔에서 CORS 오류가 재현되지 않는다.
- 기존 브랜드 선택 UX가 동일하게 동작한다.

## 의존성
- Task 3 KCAR 브랜드 프록시 구현
- Task 4 API 검증 문서

## 리스크
- 상대 경로/절대 경로 설정 불일치로 개발/배포 동작 차이 가능.
