# Sub Task Plan: QA 및 접근성 검증

## 목표
- 4단계 카테고리 기능이 요구사항대로 동작하는지 회귀/접근성 관점에서 점검한다.

## 작업 범위
- 기능 QA
  - 단계별 API 호출 및 상태 전이
  - 상위 선택 변경 시 하위 초기화
  - 오류/재시도 동작
- 접근성 QA
  - 키보드 포커스 이동
  - `aria` 속성(`role`, `aria-live`, `aria-pressed`) 검증
- 반응형 QA
  - 데스크톱/모바일 레이아웃 확인

## 검증 시나리오
1. 브랜드 선택 직후 2차 목록 로드 확인
2. 2차 선택 시 3차 로드, 기존 3/4차 선택 초기화 확인
3. 3차 선택 시 4차 로드, 기존 4차 선택 초기화 확인
4. API 실패 시 오류 문구/재시도 버튼 동작 확인
5. 빈 데이터 시 단계별 빈 상태 문구 확인
6. 키보드만으로 1~4차 선택 가능 여부 확인

## 완료 기준
- 기능 정의서의 사용자 시나리오 1~7 단계가 재현된다.
- 주요 오류 케이스에서 사용자 안내가 명확하다.
- 모바일에서도 4단계 선택 UI가 깨지지 않는다.

## 의존성
- Task 2, Task 3, Task 4 완료

## 리스크
- 실제 QA 환경과 운영 브라우저 차이로 재현 불일치가 발생할 수 있다.
