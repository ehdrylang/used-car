# Sub Task Plan: KCAR 2/3/4차 선택 UI 및 상호작용 구현

## 목표
- 기존 브랜드 선택 UI를 유지하면서 2/3/4차 선택 영역을 추가해 4단계 카테고리 탐색을 완성한다.

## 작업 범위
- 좌측 패널 UI 확장
  - 2차(모델군), 3차(세부 모델), 4차(등급/트림) 섹션 추가
- 단계별 상태 UI
  - 로딩/오류/빈 데이터/정상 목록
- 선택 상태 시각화
  - active 버튼, aria-pressed 반영
- 우측 요약 UI 확장
  - 1~4차 선택값 및 `count` 표시
  - 미선택 단계 가이드 문구 표시

## 구현 순서
1. 2/3/4차 리스트 렌더 함수 추가
2. 단계별 클릭 이벤트 바인딩 연결
3. 단계별 상태 메시지 렌더링 분기 추가
4. 우측 선택 요약 카드 확장
5. 모바일 레이아웃 대응 스타일 보강

## 완료 기준
- 한 화면에서 1~4차를 순서대로 선택할 수 있다.
- 어떤 단계든 로딩/오류/빈 상태가 구분되어 표시된다.
- 선택 상태가 버튼 스타일과 요약 카드에 일관되게 반영된다.

## 의존성
- Task 3 상태/데이터 흐름 확장

## 리스크
- 목록 길이가 길어질 때 가독성과 스크롤 동선이 나빠질 수 있다.
